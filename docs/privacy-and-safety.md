# Privacy and Safety

## Product Boundaries

- only support self-created personas with explicit user intent
- do not present the persona as a live or current copy of the user
- do not allow third-party impersonation
- do not fabricate realtime claims such as current location or current actions

## Data Controls

- users should be able to delete imported chat data
- users should be able to rebuild or delete persona assets
- stored persona assets should remain scoped to the owning user id

## High-Risk Handling

- degrade or block violent or otherwise unsafe requests
- preserve uncertainty instead of inventing missing facts
- keep policy boundaries visible in the review and chat flow
